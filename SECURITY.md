# Security Policy

This repository contains creative and documentation assets (story
materials, scripts, audio, and image files) rather than executable
application code. There is no running software surface in this
repository to report traditional vulnerabilities against.

## Reporting a Concern
If you discover:
- A file that appears corrupted, malicious, or tampered with, or
- Any GitHub Actions workflow behaving unexpectedly,

please open an issue describing the concern, or contact the repository
owner directly rather than filing a public issue if the concern is
sensitive.

## Scope
This policy covers the integrity of files hosted in this repository. It
does not cover the creative content itself, which is governed by
[`LICENSE`](LICENSE).
